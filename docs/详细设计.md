需求及设计文档

项目概述：我现在要做一个基于 Web 的文本/文件中转站，用于跨设备、跨系统传输文本内容及文件（可能是超大文件，但不能超过5GB），项目名称为 DropX。

功能描述：用户输入域名 dropx.happy365.day 进入系统，页面有两个显眼的标签页：“创建”、“提取”，默认选中“创建”。选中“创建”时，下方展示内容类型的下拉选项，包括“文本”和“文件”，选中“文本”选项下方出现文本框，可以输入内容，点击“存入”按钮进行保存，保存完成弹窗之后显示5位数字的提取码，提示“提取码用于提取文件，只展示一次，请牢记”，并提供复制功能。选中“文件”标签时，展示文本上传区域，点击上传文件。控制文件大小不超过5GB。每次只能上传一个文件，文件上传完成后弹窗展示4位数字提取码，提示“提取码用于提取文件，只展示一次，请牢记”，并提供复制功能。
内容类型下拉选项下方展示提示信息：“存入内容仅保留24小时，请及时提取。”。

后端文件存储采用minIO。


数据库设计：

采用 MySQL 数据库。

建表语句如下：
create table dx_content
(
    content_id int auto_increment not null,
    extract_code char(5) not null,
    content_type varchar(10) not null,  -- text or file
    status char(1) not null,  -- Y or N
    expire_at datetime not null,
    create_at datetime not null default current_timestamp,
    primary key (content_id)
);

create index dx_content_extract_code on dx_content(extract_code);


后端设计：

后端接口统一返回：
{
    "code": 0 | -1,
    "message": "处理成功" | "处理失败",
    "data": 返回的数据
}

类名：ContentController

接口：

POST /files

接收文件，生成提取码 5 位数字的 extract_code 提取码，提取码不能与数据库中还有效的文件的提取码相同，设置 contentType = "file"，设置过期时间 expire_at 为 24 小时后，将文件上传至 minIO，获得文件路径，设置status = 'Y'，存入数据库。并返回提取码和成功消息。

POST /text

接收文本，设置 contentType="text"，其他处理逻辑同上，将数据存入数据库。

GET /files/{extractCode}

从数据库查询对应的 extract_code 且 status = 'Y' 的文件，得到文件路径将文件下载到前端，如果不存在这样的数据，则返回提示文件不存在或已过期。

GET /text/{extractCode}

从数据库查询对应的 extract_code 且 status = 'Y' 的文本，得到文件路径将文件下载到前端，如果不存在这样的数据，则返回提示文本内容不存在或已过期。
